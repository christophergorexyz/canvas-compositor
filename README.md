## canvas-compositor
A light, performant compositing engine for the HTML5 canvas

### Getting started

You should be able to include the compiled files from the dist directory in any HTML page, which will add the `CanvasCompositor` namespace to your global scope. I like to assign it to a shorthand `cc`. 

You can then start using the CanvasCompositor's scene and graphics APIs by instantiating it with a canvas: 

```
let cc = CanvasCompositor;
let _cc = new cc.Compositor(document.getElementById('myCanvas'));
```

Our `_cc` variable exposes classes for drawing primitives (currently including `Rectangle`, `Circle`, `Ellipse`, `Polygon`, `Bezier`, `Picture`, and `Text`) which are used to build `Composition`s.

The `_cc` variable will have a `scene` property. If you are only drawing pixels to canvas, this property is unnecessary, but if you intend to have any kind of animated or layered interactions, this object will be incredibly important. 

The `scene` property is the entry point to your scene graph. It is an instance of `Composition` (which is also exposed, and can be extended freely). `Composition`s have `children` - the order of the `scene`'s `children` determines the order in which they are drawn. 

`Composition`s like `scene` can have children added to them through the `addChild` or `addChildren` method. The children can be `Composition`s or any other inheritor of the `PrimitiveComponent` class. 

`Composition`s and `Component`s, in conjunction with shape components, comprise an implementation of the [Composite Pattern](http://en.wikipedia.org/wiki/Composite_pattern) - thus, "canvas-compositor".

[View the demo here](https://christophergorexyz.github.io/canvas-compositor/demo/)

### Composition bounds modes

`Composition` supports two bounds behaviors:

- **`fixed`** (default): the composition keeps its current `width`/`height`; children can move beyond local bounds but pixels outside those bounds are clipped.
- **`auto-expand`**: the composition recalculates bounds from child positions/sizes each render pass and expands as needed so moved children remain visible.

You can configure this per composition:

```js
const group = new CanvasCompositor.Components2d.Composition(260, 180, { x: 100, y: 100 });
group.boundsMode = 'auto-expand';
```

For the root scene, you can also resize the target HTML canvas when scene bounds expand:

```js
const cc = new CanvasCompositor.Compositor(document.getElementById('myCanvas'));
cc.scene.boundsMode = 'auto-expand';
cc.scene.autoResizeTargetCanvas = true;
```

Notes:

- Auto-expand introduces extra bounds calculations each render of that composition.
- Child motion that crosses negative local coordinates is preserved by internal content offsets.
- If root auto-resize is enabled, canvas dimensions can change while interacting (which affects layout and pointer mapping in host pages).

### Demo debugging tools

The demo includes optional debugging controls:

- `Draw bounds` overlays a red bounding box for every component and composition.
- `Group auto-expand bounds` toggles auto-expanding bounds for the nested demo composition.
- `Auto-resize root canvas` toggles root scene auto-expand + canvas resizing.
